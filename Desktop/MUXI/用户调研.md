# 用户调研

* 为用户归类，选择典型的代表

  > * 高管（宏观层面）
  > * 经理（管理层面）
  > * 操作人员（具体层面）

* 设定目标，使问题更聚焦

* 明确调研的形式

  > * 眼动实验
  > * 可用性测试
  >   1. 资源准备
  >   2. 任务设计
  >   3. 用户招募
  >   4. 测试执行
  >   5. 报告呈现
  > * 用户访谈
  > * A/B测试
  > * 问卷调查（避免封闭性问题 金字塔结构 漏斗结构 菱形）
  > * 焦点小组
  > * 参与式设计

* 用户访谈

* 问卷调查

* 情景调查

* 安排问题的顺序

* 要研究用户的行为，更要研究用户的想法
